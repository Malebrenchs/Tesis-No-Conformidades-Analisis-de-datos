# QIPO-014: Vinculación con Marcos de Referencia (PMI, ISO, Análisis de Datos)

## Q
¿Cómo se integran formalmente los marcos metodológicos aplicables (PMI, ISO 9001, análisis de datos, mejora continua) en la propuesta de gestión de no conformidades y en el procedimiento desarrollado?

## I
Propuesta metodológica diseñada en QIPO-013.  
Resultados del análisis exploratorio (QIPO-012).  
Buenas prácticas internacionales sobre gestión documental, gestión del conocimiento, control de calidad y ciencia de datos.

## P
Se identifican los marcos de referencia que sustentan técnica y metodológicamente el procedimiento de análisis y la propuesta de gestión de no conformidades, estableciendo su aporte específico y la razón de su pertinencia.

### Tareas prácticas

#### 1. Identificar marcos normativos y metodológicos aplicables
- PMI (PMBOK)  
- ISO 9001:2015  
- CRISP-DM (ciencia de datos)  
- Mejora continua (PHVA)  
- Data Governance (principios de gestión de datos)

#### 2. Establecer correspondencias entre el procedimiento desarrollado y cada marco
- **PMI:**  
  - Gestión de las comunicaciones → proceso de emisión, respuesta y cierre de NC.  
  - Gestión del conocimiento → registro histórico de NC y aprendizajes.  
  - Control de calidad → confirmación de corrección y cierre formal.  
  - Integración → consolidación de información para toma de decisiones.

- **ISO 9001:**  
  - Información documentada → estructura tabular normalizada para NC.  
  - Control del producto no conforme → flujo de identificación, corrección y verificación.  
  - Trazabilidad → uso de identificadores únicos (id_nc, id_evento).

- **CRISP-DM:**  
  - Comprensión del negocio → diagnóstico inicial del archivo.  
  - Preparación de los datos → limpieza, transformación, normalización.  
  - Modelado y evaluación → análisis exploratorio y generación de KPIs.  
  - Implementación → propuesta metodológica y ciclo de análisis periódico.

- **PHVA / Kaizen:**  
  - Plan → diseño del modelo de datos y metodología.  
  - Do → ejecución de limpieza y análisis.  
  - Check → revisión de indicadores.  
  - Act → mejoras continuas al proceso de gestión de NC.

- **Data Governance:**  
  - Consistencia → normalización de campos y reglas de integridad.  
  - Calidad de datos → control de fechas, valores y estructura.  
  - Accesibilidad → uso de formatos procesables como CSV o Excel limpio.

#### 3. Redactar la integración formal
- Consolidar cómo cada marco aporta justificación técnica.  
- Explicar por qué su aplicación es pertinente para obras de construcción.  
- Integrar esta justificación como anexo metodológico o sección del capítulo teórico.

### Herramientas a utilizar
- Documentación oficial:
  - PMBOK (Project Management Institute).  
  - ISO 9001:2015.  
  - Modelo CRISP-DM.  
- Documentos de soporte de gestión documental y data governance.  
- Markdown para redactar la vinculación formal dentro del repositorio.

### Marcos que justifican las acciones
- **Estándares internacionales** (PMI, ISO) → legitimidad metodológica.  
- **Metodologías de ciencia de datos** (CRISP-DM) → justificación del proceso analítico.  
- **Gestión de calidad** → trazabilidad y control de NC.  
- **Gestión de datos** → estructura, consistencia y confiabilidad de la información.  
- **Mejora continua** → implementación práctica y sostenibilidad del proceso propuesto.

## O
Se obtiene una articulación clara y formal entre el procedimiento desarrollado y los marcos de referencia internacionales, fortaleciendo la validez técnica, académica y metodológica del trabajo.
